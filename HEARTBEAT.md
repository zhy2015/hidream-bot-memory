# HEARTBEAT.md - 生产环境巡检协议

## 核心模式：监控 → 清理 → 汇报

视觉工坊作为内容生产节点，核心任务是确保存储空间充足、渲染队列健康、成品及时交付。

## 定时后台巡检任务 (每 6-12 小时执行一次)

### 1. 渲染队列监控 (Render Queue Watchdog)
- **检查**: 是否有挂起超过 1 小时的生成任务（可能是 API 超时或脚本卡死）。
- **动作**:
  - 如果发现僵尸任务，强制终止并标记为 Failed。
  - 尝试重启未完成的高优先级任务。

### 2. 磁盘空间清理 (Garbage Collection)
- **目标**: `downloads/` 和 `output/` 目录。
- **规则**:
  - **保留**: 最近 24 小时内的素材。
  - **删除**: 超过 24 小时的临时中间文件 (temp_*.mp4, *.png)。
  - **归档**: 重要的成品视频建议提示用户手动归档，或者上传到云端（如果有配置）。
- **阈值**: 如果总占用超过 5GB，立即触发激进清理。

### 3. 交付确认 (Delivery Check)
- **检查**: 是否有生成成功但未发送邮件的文件（比如因网络波动发送失败）。
- **动作**: 重新触发 `qqmail` 发送流程。

## 汇报格式

```
🎬 [生产日报]
───────────────────────
✅ 今日产出: [N] 个视频
🗑️ 清理空间: [N] MB
⚠️ 异常告警: [无 / 具体错误]
───────────────────────
状态: 渲染节点正常待机中
```

## 执行节奏

- **忙碌时**: 每次任务完成后自动触发简易检查。
- **闲时**: 每天早晚各一次全量巡检。
